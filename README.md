# 銘柄管理ダッシュボード（Streamlit版）

Obsidian の Dataview / Templater で管理していた株式ウォッチリストを、
ブラウザで動く Web アプリに置き換えるための Streamlit アプリです。

## 元のコードからの変更点

- `time.sleep(1)` でエラーになっていたバグを修正（`import time` 漏れ）
- 「📋 一覧・編集」タブに **直接編集・行の削除・新規行追加** ができる表（`st.data_editor`）を追加
- CSV ダウンロードボタンを追加（バックアップやExcelで開きたい時用）
- 「📊 分析」タブを新規追加
  - セクター別の円グラフ
  - ステータス別の銘柄数
  - 銘柄別 PER の比較グラフ
  - 登録銘柄数・保有中の数・平均PERなどのサマリー
- 同じティッカーを再登録しようとした際に警告を表示（誤って二重登録するのを防ぐ）
- 「📝 新規銘柄登録」タブに **ティッカーコードからの自動取得機能** を追加（Yahoo!ファイナンス / yfinance経由）
- 「🧮 売上CAGR計算」タブを新規追加
  - ティッカーからの自動取得（yfinanceの決算データを使用）
  - IR資料などの数値を手入力して計算するモード（小型株などYahoo側データが薄い場合の保険）
- 「📑 個別銘柄レポート」タブを新規追加
  - 銘柄ごとにMarkdown形式の詳細レポート（決算所感、SWOT分析、AIが生成したレポートの貼り付けなど）を保存
  - 保存前に自動バックアップ（`reports/_backup/`）、個別・全件のダウンロード機能つき
  - **AI分析用プロンプトのコピー機能**：SWOT分析・キャッシュフロー表・目標株価3シナリオ・Fスコアなどを指示するプロンプトを、選択中の銘柄データ付きでワンクリックコピーできる
  - **決算スケジュール（カウントダウン）**：yfinance経由で次回決算発表予定日を取得し、残り日数を表示
  - **基本データカード**：選択中の銘柄のステータス・PER・売上5y CAGR・ネットキャッシュなどを一覧表示

## 個別銘柄レポートタブについて

- レポートは銘柄ごとに `reports/＜ティッカー＞.md` として保存されます。
- **⚠️ Streamlit Community Cloudでは要注意**：`portfolio_data.csv`と同様、`reports/`フォルダもCloud環境では再起動・再デプロイで消える一時的な領域です。決算メモなど大事な内容は、タブ内の「📦 全レポートをZIPでダウンロード」で定期的に手元に保存することを強くおすすめします。
- 上書き保存する際は、保存前の内容が自動的に`reports/_backup/`にタイムスタンプ付きで退避されます（誤って消してしまった時の保険）。
- **AI分析用プロンプト**：銘柄を選択すると、SWOT分析・3シナリオの目標株価・ピオトロスキーFスコアなどを指示する分析プロンプトが、その銘柄の登録データ（セクター・PER・売上5y CAGRなど）付きで自動生成されます。コードブロック右上のコピーアイコンでコピーし、ChatGPT/Gemini/Claudeなどに貼り付けて使ってください。プロンプト本文を直接書き換えたい場合は、コード中の`ANALYST_PROMPT_TEMPLATE`を編集してください。
- **決算スケジュール**：「📅 決算スケジュールを取得」ボタンを押すと、yfinance経由でYahoo!ファイナンスから次回決算発表予定日を取得し、残り日数を表示します。ただしYahoo側がまだ日程を公表していない銘柄や、データが古い銘柄では取得できない・ズレることがあります。

## 売上CAGR計算タブについて

- 自動取得モードでは、Yahoo!ファイナンスの年次決算データ（Total Revenue）を使ってCAGRを計算します。
- **注意：Yahoo!ファイナンスの無料データは通常「直近4期分」程度しか提供されません。** そのため「5年」分のデータが取れず、実際には3〜4年分のCAGRになることが多いです。アプリ側でも実際に何年分のデータを使ったか（「実質 約N年 CAGR」）を表示するので、必ず確認してください。
- 小型株・新興企業などでYahoo側のデータが取得できない場合は、「手入力で計算」モードで会社の決算短信・IR資料の数値を直接入力してください。
- 計算結果は `st.code` のコピーボタン付きで表示されるので、そのまま「新規銘柄登録」タブの売上5y CAGR欄に貼り付けられます。

## 自動取得機能について

ティッカーコード（例: `7974`）を入力して「🔍 銘柄情報をWeb上から自動取得」を押すと、
Yahoo!ファイナンスから以下の項目を取得してフォームに反映します。

| 項目 | 自動取得 | 補足 |
|---|---|---|
| 銘柄名 | ◯ | |
| PER | ◯ | 直近のトレーリングPER |
| ネットキャッシュ | ◯（簡易判定） | (現金−負債)÷時価総額 の比率から「潤沢／普通／マイナス」を簡易判定。正式な分析の代わりにはならないので参考値としてご利用ください |
| セクター | ✕（手動） | Yahoo側の業種分類が東証の業種分類とうまく一致しないため、誤った自動分類を避けて手動選択にしています |
| 売上5y CAGR / 売上予想 | ✕（手動） | 決算分析が必要なため対象外。ここはご自身の分析の価値が出る部分です |

**注意点**
- 数字だけのコード（`7974`など）は自動的に東証コード（`7974.T`）として検索します。米国株などは `AAPL` のようにそのまま入力すれば取得できます。
- yfinanceはYahoo!ファイナンスの非公式なデータ取得方法のため、Yahoo側の仕様変更で突然動かなくなることがあります。その場合はエラーメッセージが表示されるので、その時はお知らせください。
- 取得した数値は鵜呑みにせず、必ず目で確認してから登録してください（特にPERは決算発表のタイミングで実態とズレることがあります）。

## 動かし方（自分のPCで）

1. Python 3.9以上がインストールされていることを確認
2. このフォルダに `dashboard_app.py` と `requirements.txt` を置く
3. ターミナル（コマンドプロンプト）で以下を実行

```bash
pip install -r requirements.txt
streamlit run dashboard_app.py
```

4. 自動的にブラウザが開き、`http://localhost:8501` でアプリが表示されます
5. データは同じフォルダ内の `portfolio_data.csv` に自動保存されます

※ 一度終了してもう一度 `streamlit run dashboard_app.py` を実行すれば、前回保存したデータがそのまま読み込まれます。

## どこからでもアクセスできる「本当のWebアプリ」にしたい場合

上記はあくまで自分のPC上で動くものです（PCを閉じるとアプリも止まります）。
スマホからもアクセスしたい、家族や他のPCからも見たい、という場合は
**Streamlit Community Cloud**（無料）にデプロイするのが一番簡単です。

1. このコードを GitHub のリポジトリにアップロードする（`dashboard_app.py` と `requirements.txt`）
2. https://share.streamlit.io にアクセスし、GitHubアカウントでログイン
3. 「New app」→ 対象のリポジトリと `dashboard_app.py` を選択 → Deploy
4. 数分後に `https://〇〇.streamlit.app` のような公開URLが発行される

**注意点**：Community Cloud上では `portfolio_data.csv` の保存が一時的（再起動で消える場合がある）なため、
本格的に複数人で・長期間使う場合は、データベース（例: Google Sheets連携やSupabase、SQLiteなど）への
切り替えをおすすめします。必要であればそちらの実装もお手伝いできます。
